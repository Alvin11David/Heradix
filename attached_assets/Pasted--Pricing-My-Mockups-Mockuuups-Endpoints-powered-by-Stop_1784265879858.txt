Render Mockup

Request
Body

application/json

application/json
Parameters for rendering a single mockup.

mockup
string
required
Mockup ID. Use the ID received from the Mockup Listing endpoint, or find your desired mockup and copy its ID from the website.

Example:
YXpoZdyRLAKQHgQA
size
integerobject

one of: integer
If you provide a number, it will scale the mockup by its longer edge to match the size.Show all...

format
string
Use to override the export file format. We recommend using image/webp if your application is ready to handle this format, as it has the best compression ratio. If you don't specify any format, we automatically select PNG for transparent mockups, and JPEG for the rest.

Allowed values:
image/png
image/jpeg
image/webp
Default:
null
trim
boolean
Transparent pixels in the rendered mockup will be automatically trimmed. Set to false to prevent this behavior. This option has no effect on mockups without transparency.

Default:
true
contents
array[Content]
required
Content to be placed in placements. The order of the array and its length must match the array of placement definitions from the Mockup object.

>= 1 items
type
string
required
Allowed values:
image
screenshot
Default:
image
url
string<uri>
required
URL of the content which must match the selected type.

Example:
https://assets.mockuuups.com/mo/image/upload/sample-media.jpg
crop
object
Optionally adjust which part of the content image will be placed into the mockup.Show all...

destination
string
Destination controls how you receive your mockup.Show all...

Allowed values:
response
cdn
custom
Default:
response
mode
string
Mode controls whether the POST request waits with its response until the mockup has fully finished rendering.Show all...

Allowed values:
sync
async
Default:
sync
storage
object
If your destination is custom, you need to include this in order to specify which storage to use.

id
string
External storages need to be configured in the Developer Dashboard first, which will give you a storage connection ID that can be used here. Required if the destination is custom.

Default:
folder
string
Optional folder name (used with Cloudinary).

Default:
cdn
object
If your destination is cdn, you can configure it here.

expiration
string
Automatically delete the file from the CDN. You can set this to a date in the future, or pass a relative time like "1 day" or "2 weeks" (using syntax of systemd.time). If not specified, defaults to "1 day". Use "never" for permanent storage (requires paid plan).

Default:
1 day
Example:
1 day
webhook
string<uri>
Optionally add a URL which we will call when the render is completed, whether it's a success or an error. We will POST to this URL with a JSON body with details of the render.

Default:
meta
string
You can store any short arbitrary string in this field. It will be saved and sent in any subsequest requests for this render, and in the webhook after it finishes processing.Show all...

Default:
xmp
boolean
If true, the rendered mockup will include XMP metadata with license information confirming the image is allowed to be used commercially.

Default:
true
